# 数字孪生三维场景优化需求文档

## 主标题
项目基础定义

## 项目名称
数字孪生三维场景优化

## 项目目标
对已有数字孪生平台进行界面优化，输出新的前端页面，用于前端展示和设计确认。

## 项目性质
纯前端展示型页面，不接数据，不做真实业务处理，不做后台功能。

---

## 页面结构
8 个固定区域 + 顶部三列布局

### 开发重点
- **结构清晰**: 整体页面按 8 个区域固定拆分，避免随意改动布局
- **使用目的**: 原型展示
- **核心结论**: 先确定框架，再逐个区域补细节交互，适合多页面逐步推进
- **设计核心**: 中间以三维场景为核心，左侧放对象，右侧放属性，符合编辑型产品习惯
- **工具栏位置**: 工具栏必须放在三维场景区内部，位于场景上方区域，而不是整个页面最上方
- **底部信息**: 底部单独承载三角面、坐标、状态等杂项信息，减少对主操作区的干扰
- **顶部布局**: 顶部采用三列布局 - 左侧品牌区、中间Tab导航区、右侧操作按钮区

### 页面定位
这是一个展示型原型页面，不是完整业务系统。
- 页面重点在于布局、区域边界、视觉层级、基本交互状态展示
- 所有内容可使用静态假数据、占位字段、示意模型、示意状态
- 允许做标签切换、选中态、hover 态、面板折叠等基础前端演示
- 不要求真实三维渲染能力，可使用占位画面模拟三维场景区

> **说明**: 开发时应把重点放在"页面结构稳定、区域关系清晰、可持续扩展"上，而不是过早追求复杂功能或真实数据联动

### 行动建议 / 下一步

#### 建议开发顺序
1. **第一步：搭框架** - 先按 8 个固定区域完成页面骨架，不要在此阶段扩展复杂逻辑
2. **第二步：补内容** - 给每个区域填充静态文案、按钮、列表、表单、场景占位图等内容
3. **第三步：补状态** - 增加 tab 切换、选中态、高亮态、hover 态、折叠态等基础交互效果
4. **第四步：补视觉** - 统一颜色、间距、圆角、阴影、边框和信息层级，形成完整原型页面

## 页面整体需求

### 全局设计原则

| 维度 | 需求说明 |
|------|----------|
| 页面风格 | 现代、清晰、专业、偏控制台/编辑器风格；不做复古、报刊、花哨装饰风 |
| 布局方式 | 采用清晰的多区域布局，区域边界明确，可直接看出每个模块的职责 |
| 信息层级 | 标题、tab、对象、场景、属性、底部信息要有明显层级区分，重点区域是三维场景区 |
| 交互方式 | 仅保留前端演示级交互，例如切换、悬停、选中、展开折叠、按钮状态，不做真实业务逻辑 |
| 数据来源 | 全部为静态假数据、假文案、示意字段 |
| 技术范围 | 只做前端页面，可使用 Vue 3 + Vite；不依赖后端接口 |
| 开发原则 | 优先保证结构稳定、模块独立、便于后续替换内容 |

## 8 个区域详细需求

### 平台核心结构说明

| 区域名称 | 位置 | 主要内容 | 展示要求 |
|----------|------|----------|----------|
| 标题展示区 | 顶部左侧 | PCI 佳都品牌、三维场景编辑器、两江影视城站AFC智能运维 | 突出页面身份信息，简洁明确 |
| 状态 Tab 切换区 | 顶部中间 | 场景/模型/状态/资产/工具 主Tab及子Tab | 支持视觉切换，当前tab高亮 |
| 对象区 | 左侧 | 各Tab对应的对象列表、树结构、设备模型 | 支持层级展示、选中态、设备拖拽 |
| 默认设置区 | 顶部右侧 | 场景信息开关、预览、保存、帮助菜单 | 展示操作按钮组 |
| 工具栏区 | 三维场景区内部上方 | 新增、撤回、重做、变换工具、步长设置、默认视图 | 做成悬浮工具条 |
| 属性配置区 | 右侧 | 选中对象的属性配置、三维视图配置、材质库、动画配置等 | 以表单卡片方式展示 |
| 三维场景区 | 中间 | 三维场景主画面、背景图片、设备标注 | 背景图片展示，可拖拽放置设备 |
| 杂七杂八信息区 | 底部 | Render、WebGL2、FPS、三角面、坐标、场景状态 | 做成底部信息栏 |

#### 主Tab

## 各区域交互需求

### 仅限前端展示型交互

#### 标题展示区
- 显示项目标题与工具名称
- 可展示"原型设计中""当前页面"等状态标签
- 不需要真实跳转或下拉逻辑

#### 状态 Tab 切换区
- 点击 tab 时切换高亮状态
- 可同步改变对象区标题文案和工具栏文案
- 不需要真实内容接口切换，可用静态切换效果模拟

#### 对象区
- 支持列表项选中效果
- 支持树结构展开/收起演示
- 搜索框只做样式展示或简单前端筛选
- 支持设备拖拽到三维场景

#### 默认设置区
- 按钮包括保存、导出、预览、重置等
- 只需 hover、active、disabled 等样式
- 不需要实际文件导出和保存
- 帮助菜单为点击触发下拉菜单

#### 工具栏区
- 位于三维场景区内部顶部
- 点击工具按钮时切换选中态
- 不同 tab 可展示不同工具组，但可以用静态内容模拟

#### 属性配置区
- 选中对象后右侧显示对应属性内容
- 可演示文本输入框、开关、下拉框、坐标输入框等
- 无需表单提交，无需校验逻辑

#### 三维场景区
- 作为中心主区域，展示三维画面占位
- 可加入假模型块、点位标注、信息悬浮卡
- 支持拖拽设备放置
- 不做真实 3D 引擎，不做模型加载
- 显示背景图片

#### 杂项信息区
- 显示三角面数、坐标值、相机参数、状态提示等
- 以信息块形式集中展示
- 不需要实时刷新逻辑

### 交互边界说明
- 所有交互以"演示状态"为主
- 不涉及接口请求、鉴权、数据库、三维引擎联动
- 不涉及真实业务流程提交

## 前端开发要求

### 给 AI 或前端开发的明确约束

#### 必须做到
- 按 8 个区域实现页面结构
- 工具栏放入三维场景区内部
- 页面必须有清晰边界、卡片、分区和层级
- 页面需适合直接复制成完整前端静态页面
- 可以使用静态假数据填充所有区域
- 样式统一、现代、简洁、可读性高
- 顶部采用三列布局
- 支持设备拖拽到三维场景

#### 不要做到
- 不要接入后端接口
- 不要实现真实保存、导出、发布逻辑
- 不要使用复杂业务状态机
- 不要把工具栏放在页面外层顶部
- 不要做复古报纸风、花哨装饰风
- 不要让 AI 自由发挥更改整体布局结构

> **推荐做法**: 将每个区域拆分成独立组件，例如 TitlePanel、TabPanel、ObjectPanel、ScenePanel、SceneToolbar、PropertyPanel、FooterInfoPanel，方便后续继续扩展多页面

## 验收标准

### 用于判断页面是否达标

| 验收项 | 标准 |
|--------|------|
| 结构验收 | 是否完整体现 8 个区域，且位置关系符合要求 |
| 布局验收 | 工具栏是否明确位于三维场景区内部，而不是页面外部顶部；顶部是否为三列布局 |
| 视觉验收 | 页面是否现代、清晰、重点突出、适合展示，不杂乱 |
| 交互验收 | 是否具备基本的 tab、高亮、选中、按钮、开关、拖拽等演示效果 |
| 内容验收 | 各区域是否有对应的示意内容，而不是空白占位 |
| 范围验收 | 是否严格保持前端展示性质，没有扩展到真实数据功能 |
| 功能验收 | 资产模块、工具模块、右上功能区是否完整实现 |
